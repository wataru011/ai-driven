# 用語集

AI駆動開発で頻出する用語を簡潔に定義します。定義は1〜3文に絞り、より深い解説は関連ページへリンクしています。英字・略語を先に、続いて日本語の用語を五十音順で並べています。

::: tip 使い方
ここは辞書として引くページです。概念の体系的な理解は[LLMの仕組み](/concepts/how-llms-work)から、文脈の中で読むことをおすすめします。素朴な疑問は[FAQ](/reference/faq)、即用の型は[チートシート](/reference/cheatsheet)へ。
:::

## 英字・略語

| 用語 | 定義 |
| --- | --- |
| **Chain-of-Thought(CoT, 思考の連鎖)** | 結論の前に推論過程を段階的に書かせ、複雑な問題の正答率を高めるプロンプト技法。→ [プロンプトの基礎](/prompting/fundamentals) |
| **codemod** | コードを構文木レベルで機械的に一括変換するスクリプト/手法。大規模リファクタリングや移行で使われ、AIが生成役を担うこともある。 |
| **DPO(Direct Preference Optimization)** | 人間の選好データから直接モデルを最適化する手法。報酬モデルを別途学習するRLHFより手順が簡潔。 |
| **eval(評価)** | モデルやプロンプトの性能を、定義したタスク・データセット・採点基準で測ること。AI機能の品質を継続的に管理する土台。 |
| **Few-shot(少数例提示)** | プロンプト内に入出力の例をいくつか示して、期待する形式や挙動を学ばせる手法。例を示さない場合は Zero-shot。→ [プロンプトの基礎](/prompting/fundamentals) |
| **function calling(関数呼び出し)** | モデルが定義済みの外部関数/ツールを構造化して呼び出す仕組み。ツール使用の実装形態でエージェントの基本要素。 |
| **jailbreak(ジェイルブレイク)** | モデルの安全制約やポリシーを回避させ、本来拒否すべき出力を引き出す試み。プロンプトインジェクションの一種。→ [セキュリティ](/quality/security) |
| **LLM(大規模言語モデル)** | 大量のテキストで訓練され、次に来るトークンの確率を予測することで自然言語やコードを生成するモデル。AI駆動開発の中核。→ [LLMの仕組み](/concepts/how-llms-work) |
| **LLM-as-a-judge** | モデルの出力の良し悪しを、別のLLMに採点・判定させる評価手法。eval を人手より安価にスケールさせる。 |
| **LoRA(Low-Rank Adaptation)** | 元の重みを凍結し小さな低ランク行列だけを学習する、軽量なファインチューニング手法。PEFTの代表例。 |
| **MCP(Model Context Protocol)** | AIアプリを外部のデータソースやツールに接続するためのオープンな標準プロトコル。ツール連携の「共通規格」。→ [リソース集](/reference/resources) |
| **PEFT(Parameter-Efficient Fine-Tuning)** | モデル全体ではなく一部パラメータだけを更新して効率的に適応させる手法群の総称。LoRAはその代表。 |
| **RAG(Retrieval-Augmented Generation)** | 生成前に外部知識を検索し、その内容をコンテキストへ注入して精度と最新性を高める手法。ハルシネーション抑制に有効。→ [RAG](/concepts/rag) |
| **ReAct(Reasoning + Acting)** | 推論と行動を交互に行わせるエージェントの設計パターン。「考える→ツールを使う→観察する」を繰り返す。→ [エージェントとは](/agents/what-is-agent) |
| **RLHF(人間のフィードバックによる強化学習)** | 人間の選好データを用いてモデルの出力を望ましい方向へ調整する訓練手法。有用性・無害性の向上に使われる。 |
| **slopsquatting** | LLMが幻覚で提案しがちな「実在しないパッケージ名」を攻撃者が先回りして公開し、取り込ませるサプライチェーン攻撃。依存追加前の実在性確認が必須。→ [セキュリティ](/quality/security) |
| **TDD(テスト駆動開発)** | テストを先に書き、それを通す実装を作り、リファクタする開発手法。AIに実装させる際の検証の土台になる。 |
| **temperature(温度)** | 出力のランダムさを制御するパラメータ。低いと決定的・保守的、高いと多様・創造的。コード生成では低めが好まれることが多い。 |
| **top-k** | 確率上位 k 個の候補だけからサンプリングする手法。候補数を固定して多様性を制御する。 |
| **top-p(核サンプリング, nucleus sampling)** | 累積確率が p に達するまでの上位候補だけからサンプリングする手法。出力の多様性を制御する。 |
| **Zero-shot(ゼロショット)** | 例を一切示さず、指示だけで望む出力を得ようとする手法。Few-shot の対。 |

## 五十音順

| 用語 | 定義 |
| --- | --- |
| **エージェント(Agent)** | 目標を与えられ、ツールの使用・観察・次の行動の決定を自律的に繰り返してタスクを遂行するLLMベースのシステム。→ [エージェントとは](/agents/what-is-agent) |
| **埋め込み(エンベディング, Embedding)** | テキストなどを、意味的な近さが距離に反映される高次元ベクトルへ変換したもの。意味検索やRAGの基盤。→ [RAG](/concepts/rag) |
| **温度** | → temperature(英字の項を参照)。 |
| **ガードレール(Guardrail)** | モデルの入出力を監視・制約し、危険・不適切・ポリシー違反な振る舞いを防ぐ仕組み。→ [セキュリティ](/quality/security) |
| **ガバナンス(Governance)** | AI利用のポリシー・責任・監査を組織的に統制する枠組み。→ [導入戦略](/adoption/strategy) |
| **量子化(Quantization)** | モデルの重みを低ビット精度(例: 16ビット→4ビット)で表現し、メモリと計算コストを削減する手法。精度とのトレードオフを伴う。 |
| **グッドハートの法則** | 「ある指標が目標になると、その指標は良い指標でなくなる」という経験則。生産性測定の落とし穴。→ [生産性の測定](/adoption/metrics) |
| **グラウンディング(Grounding)** | モデルの出力を、信頼できる外部情報や事実に紐づけて根拠づけること。RAGはその代表的手段。→ [RAG](/concepts/rag) |
| **ゴールデンセット(Golden Set)** | 期待する正解をあらかじめ固定した評価用データセット。eval の基準として回帰を検出するのに使う。 |
| **コードベースインデックス(Codebase Index)** | リポジトリの内容を検索可能に索引化したもの。エージェントやRAGが関連コードを見つけるための土台。 |
| **コンテキストウィンドウ(Context Window)** | モデルが一度に処理できるトークンの最大量。入力・履歴・出力の合計がこの上限に収まる必要がある。→ [LLMの仕組み](/concepts/how-llms-work) |
| **コンテキストエンジニアリング(Context Engineering)** | 限られたコンテキストウィンドウに、必要な情報を過不足なく構成して与える技術。プロンプトより広く「何を文脈に入れるか」を設計する。 |
| **サンプリング(Sampling)** | 確率分布から次のトークンを選ぶ過程。temperature / top-p / top-k などのパラメータで挙動を制御する。 |
| **事前学習(Pre-training)** | 大量のテキストで次トークン予測を学ばせ、汎用的な言語能力を獲得させる最初の訓練段階。→ [LLMの仕組み](/concepts/how-llms-work) |
| **自己回帰(Autoregressive)** | 生成済みのトークンを次の入力に回し、1トークンずつ逐次生成する方式。多くのLLMの基本動作。 |
| **指示チューニング(Instruction Tuning)** | 指示と望ましい応答の対でモデルを追加学習し、指示に従う能力を高める手法。 |
| **自律度(Autonomy Level)** | エージェントに人間の介入なしでどこまで任せるかの度合い。タスクのリスク・可逆性・検証コストで決める。→ [エージェントとは](/agents/what-is-agent) |
| **蒸留(Distillation)** | 大きな教師モデルの振る舞いを小さな生徒モデルに学習させ、軽量・高速なモデルを作る手法。 |
| **スキャフォールド(Scaffold)** | プロジェクトや機能の骨組み(ディレクトリ・雛形・定型コード)を自動生成すること、またその生成物。 |
| **スループット(Throughput)** | 単位時間あたりに処理できる量(例: トークン/秒、リクエスト/秒)。バッチ処理や大量生成で重要。 |
| **セマンティック検索(Semantic Search)** | キーワード一致ではなく、埋め込みの意味的近さで関連文書を探す検索。RAGの検索段階を担う。→ [RAG](/concepts/rag) |
| **仕様駆動開発(Spec-Driven Development)** | 検証可能な仕様を先に固め、それを基準にAIへ実装させ仕様で受け入れ判定する進め方。手戻りを減らす。→ [仕様駆動開発](/advanced/spec-driven) |
| **システムプロンプト(System Prompt)** | モデルの役割・方針・制約をあらかじめ設定する、会話の前提となる指示。出力全体の振る舞いを方向づける。→ [プロンプトの基礎](/prompting/fundamentals) |
| **チェイン・オブ・ソート** | → Chain-of-Thought(英字の項を参照)。 |
| **ツール使用(Tool Use)** | モデルが、定義された外部関数やツール(検索・実行・API呼び出し等)を呼び出して能力を拡張する仕組み。function calling として実装される。 |
| **トークン(Token)** | モデルがテキストを処理する最小単位。単語より細かい部分文字列で、課金やコンテキスト長はトークン数で数える。→ [LLMの仕組み](/concepts/how-llms-work) |
| **トークナイザ(Tokenizer)** | テキストをトークン列へ分割し、IDへ変換する仕組み。言語やコードによって分割の仕方とトークン数が変わる。 |
| **ハルシネーション(Hallucination)** | モデルがもっともらしいが事実に反する、あるいは存在しない情報(架空のAPI・関数等)を生成する現象。検証が不可欠な理由。→ [ハルシネーション](/quality/hallucination) |
| **ファインチューニング(Fine-tuning)** | 事前学習済みモデルを特定タスクやドメインのデータで追加学習し、振る舞いを専門化させること。LoRAは軽量な一手法。 |
| **文脈長(Context Length)** | モデルが扱えるトークン長。コンテキストウィンドウとほぼ同義で、長いほど一度に多くの情報を渡せる。→ [LLMの仕組み](/concepts/how-llms-work) |
| **プロンプトインジェクション(Prompt Injection)** | 信頼できない入力に紛れた指示でモデルを乗っ取り、意図しない操作をさせる攻撃。エージェントの主要リスク。→ [セキュリティ](/quality/security) |
| **プロンプトエンジニアリング(Prompt Engineering)** | モデルから望む出力を引き出すために、指示・文脈・例・制約を設計する技術。→ [プロンプトの基礎](/prompting/fundamentals) |
| **プロンプトキャッシュ(Prompt Caching)** | 繰り返し使う長い前置き(システムプロンプト・文脈)をキャッシュし、コストとレイテンシを下げる仕組み。 |
| **ベクトルDB(Vector Database)** | 埋め込みベクトルを格納し、類似度で高速検索できるデータベース。セマンティック検索とRAGの基盤。→ [RAG](/concepts/rag) |
| **ヒューマンインザループ(Human-in-the-Loop)** | 自動処理の要所に人間の確認・承認を組み込む設計。高リスク操作の安全網になる。→ [エージェントとは](/agents/what-is-agent) |
| **マルチエージェント(Multi-Agent)** | 役割の異なる複数のエージェントが協調・分担してタスクを進める構成。計画役・実装役・検証役などに分ける。 |
| **マルチモーダル(Multimodal)** | テキストに加え画像・音声などの複数モダリティを入出力として扱える能力。スクリーンショットや図の理解に使う。 |
| **レイテンシ(Latency)** | 要求してから応答が返るまでの遅延。対話的な用途では特に体感に直結する。 |

## 関連用語の地図

用語同士の関係を一望すると理解が早まります。

- **生成の制御**: temperature / top-p / top-k / サンプリング — 出力のランダムさと多様性を調整。
- **品質の担保**: グラウンディング / RAG / ガードレール / eval / LLM-as-a-judge / ゴールデンセット — 誤りを抑え、出力を事実と方針に縛り、性能を測る。
- **能力の拡張**: ツール使用 / function calling / MCP / エージェント / ReAct / マルチエージェント — 単発生成から自律的な作業へ。
- **モデルの最適化**: 事前学習 / 指示チューニング / RLHF / DPO / ファインチューニング / LoRA / PEFT / 蒸留 / 量子化 — 専門化・整列・軽量化。
- **入力の設計**: プロンプトエンジニアリング / コンテキストエンジニアリング / Few-shot / Zero-shot / Chain-of-Thought / システムプロンプト / プロンプトキャッシュ — 与え方で出力の質を引き上げる。
- **検索の基盤**: トークン / トークナイザ / 埋め込み / ベクトルDB / セマンティック検索 / コードベースインデックス — 文脈を見つけて渡す土台。
- **安全と委任**: プロンプトインジェクション / jailbreak / slopsquatting / ヒューマンインザループ / 自律度 / ガードレール — 任せる範囲と守りの設計。
