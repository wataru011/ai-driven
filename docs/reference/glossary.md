# 用語集

AI駆動開発で頻出する用語を簡潔に定義します。定義は1〜3文に絞り、より深い解説は関連ページへリンクしています。アルファベット略語・英字を先に、続いて日本語の用語を五十音順で並べています。

::: tip 使い方
ここは辞書として引くページです。概念の体系的な理解は[LLMの仕組み](/concepts/how-llms-work)から、文脈の中で読むことをおすすめします。
:::

## 英字・略語

| 用語 | 定義 |
| --- | --- |
| **LLM(大規模言語モデル)** | 大量のテキストで訓練され、次に来るトークンの確率を予測することで自然言語やコードを生成するモデル。AI駆動開発の中核。→ [LLMの仕組み](/concepts/how-llms-work) |
| **LoRA(Low-Rank Adaptation)** | 元のモデル重みを凍結し、小さな低ランク行列だけを学習することで、低コストにファインチューニングする手法。完全な再学習より軽量。 |
| **MCP(Model Context Protocol)** | AIアプリケーションを外部のデータソースやツールに接続するためのオープンな標準プロトコル。ツール連携の「共通規格」。→ [リソース集](/reference/resources) |
| **RAG(Retrieval-Augmented Generation)** | 生成前に外部知識(ドキュメント・コードベース等)を検索し、その内容をコンテキストに注入して回答精度と最新性を高める手法。ハルシネーション抑制に有効。 |
| **ReAct(Reasoning + Acting)** | 推論(Reasoning)と行動(Acting)を交互に行わせるエージェントの設計パターン。「考える→ツールを使う→観察する」を繰り返してタスクを進める。 |
| **RLHF(人間のフィードバックによる強化学習)** | 人間の選好データを用いてモデルの出力を望ましい方向へ調整する訓練手法。有用性・無害性の向上に使われる。 |
| **top-p(核サンプリング, nucleus sampling)** | 累積確率が p に達するまでの上位候補だけからサンプリングする手法。出力の多様性を制御するパラメータ。 |

## 五十音順

| 用語 | 定義 |
| --- | --- |
| **エージェント(Agent)** | 目標を与えられ、ツールの使用・観察・次の行動の決定を自律的に繰り返してタスクを遂行するLLMベースのシステム。単発生成の先にある協働形態。→ [AI駆動開発とは](/guide/what-is-aidd) |
| **エンベディング(埋め込み, Embedding)** | テキストなどを意味的な近さが距離に反映される高次元ベクトルへ変換したもの。意味検索やRAGの基盤。 |
| **ガードレール(Guardrail)** | モデルの入出力を監視・制約し、危険・不適切・ポリシー違反な振る舞いを防ぐ仕組み。→ [ガバナンスと倫理](/adoption/governance) |
| **ガバナンス(Governance)** | AI利用のポリシー・責任・監査を組織的に統制する枠組み。→ [ガバナンスと倫理](/adoption/governance) |
| **グッドハートの法則** | 「ある指標が目標になると、その指標は良い指標でなくなる」という経験則。生産性測定の落とし穴。→ [生産性の測定](/adoption/metrics) |
| **グラウンディング(Grounding)** | モデルの出力を、信頼できる外部情報や事実に紐づけて根拠づけること。RAGはグラウンディングの代表的手段で、ハルシネーション抑制に効く。 |
| **量子化(Quantization)** | モデルの重みを低ビット精度(例: 16ビット→4ビット)で表現し、メモリと計算コストを削減する手法。精度と効率のトレードオフを伴う。 |
| **コンテキストウィンドウ(Context Window)** | モデルが一度に処理できるトークンの最大量。入力・履歴・出力の合計がこの上限に収まる必要がある。→ [LLMの仕組み](/concepts/how-llms-work) |
| **蒸留(Distillation)** | 大きな「教師」モデルの振る舞いを、より小さな「生徒」モデルに学習させ、軽量・高速なモデルを作る手法。 |
| **temperature(温度)** | 出力のランダムさを制御するパラメータ。低いと決定的・保守的、高いと多様・創造的になる。コード生成では低めが好まれることが多い。 |
| **チェイン・オブ・ソート(Chain-of-Thought, CoT)** | 結論の前に推論過程を段階的に書かせることで、複雑な問題の正答率を高めるプロンプト技法。 |
| **ツール使用 / 関数呼び出し(Tool Use / Function Calling)** | モデルが、定義された外部関数やツール(検索・実行・API呼び出し等)を呼び出して能力を拡張する仕組み。エージェントの基本構成要素。 |
| **トークン(Token)** | モデルがテキストを処理する最小単位。単語より細かい部分文字列で、課金やコンテキスト長はトークン数で数える。→ [LLMの仕組み](/concepts/how-llms-work) |
| **ハルシネーション(Hallucination)** | モデルがもっともらしいが事実に反する、あるいは存在しない情報(架空のAPI・関数等)を生成する現象。検証が不可欠な理由。 |
| **ファインチューニング(Fine-tuning)** | 事前学習済みモデルを、特定タスクやドメインのデータで追加学習し、振る舞いを専門化させること。LoRAは軽量な一手法。 |
| **Few-shot(少数例提示)** | プロンプト内に入出力の例をいくつか示して、期待する形式や挙動を学ばせる手法。例を示さない場合はゼロショット(zero-shot)。 |
| **プロンプトエンジニアリング(Prompt Engineering)** | モデルから望む出力を引き出すために、指示・文脈・例・制約を設計する技術。AI駆動開発の基礎スキル。 |
| **マルチエージェント(Multi-Agent)** | 役割の異なる複数のエージェントが協調・分担してタスクを進める構成。計画役・実装役・検証役などに分ける例がある。 |

## 関連用語の地図

用語同士の関係を一望すると理解が早まります。

- **生成の制御**: temperature / top-p — 出力のランダムさと多様性を調整。
- **品質の担保**: グラウンディング / RAG / ガードレール — ハルシネーションを抑え、出力を事実と方針に縛る。
- **能力の拡張**: ツール使用 / function calling / MCP / エージェント / ReAct — 単発生成から自律的な作業へ。
- **モデルの最適化**: ファインチューニング / LoRA / 蒸留 / 量子化 / RLHF — モデルを専門化・軽量化・整列させる。
- **入力の設計**: プロンプトエンジニアリング / Few-shot / Chain-of-Thought — 与え方で出力の質を引き上げる。
